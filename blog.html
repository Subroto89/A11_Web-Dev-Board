Question-1: What are the different ways to select an element in the DOM?

There are multiple ways to select an element in the DOM. Mostly used ways are ...

  • getElementById(): It gives only one element which holds the particular id given in the parentheses.
  • getElementsByClassName(): It gives an HTMLCollection of element that holds the class name given in the parentheses.
  • getElementsByTagName(): In this case, it also gives an HTMLCollection of elements based on the tag name given in the parentheses.
  • querySelector(): It gives only the first element based on the selector. Selector may be class name, id, tag name or attribute.
  • querySelectorAll(): It gives a NodeList of element based on the selector given in the parentheses.
  • getElementsByName(): Usefull for selecting form elements. It gives a NodeList of elements based on the name attribute value.

Question-2: What is the difference between innerHTML, innerText, and textContent ?

innerHTML, innerText and textContent are widely used property in the DOM for getting or setting elements or text.
innerHTML: It is used to get or set the HTML content of an element. It returns the text along with the tags.
innerText: It is used to get or set the visible text content of an element. innerText ignors HTML tags but takes CSS styles into account.
textContent: Using textContent, we can get or set text content of an element including the text of all its child elements . It ignores html tags and CSS styles.

Question-3: What is event delegation in the DOM?

Event delegation is a JavaScript technique that allows a parent element to listen for events on it's child elements. It is used where there are multiple childs elements within a parent element. Instead of attaching an addEventListener() or function to each child, the addEventListener() is attached to the parent element. When an event occurs on a child, the event propagates up to the parent and the parent element listen the event using event.target property then the event related functioning works for that particular child that clicked on. So, we can say, event delegation is the process of giving the event handling responsibility to a parent element. So that, parent can decide which child element will trigger an event when it occurs.

Question-4: What is event bubbling in the DOM?

Event bubbling is the event propagation concept in the DOM event model. When an event is occured on a child element, it propagates through the DOM tree toward it's parent then to its grandparent and so on. Thus it traverses through the DOM tree untill it reaches the root element(document).

Question-5: How do you create, add, and remove elements using JavaScript?

In JavaScript, creating, addition and removing elements can be done dynamically in a sequential steps.

Creating Element: Using createElement() method, element is created then text is set into the elemenet by the textContent property.
Adding element: appendChild() is used to add the created new element as its parents last child.
Removing element: To remove an element removeChild() is used.

The total procedure is below ...
Adding an element. //Target the parent element.
const parentElement = document.getElementById('myId'); //Creating Element const childElement = document.createElement('p');
childElement.textContent = "This is the text."; //Adding the newly created element into the parent element. parentElement.appendChild(childElement); Removing the element. parentElement.removeChild(childElement);
